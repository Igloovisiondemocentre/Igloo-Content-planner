- Implementation kind: configuration
- Route confidence: partial

### Runtime surfaces
- Assemble the experience as a saved Igloo session so an operator can relaunch it consistently.
- Use the documented 360-degree content workflow for panoramic imagery or 360 video.
- A likely supported route is the dedicated YouTube 360 workflow, or a hosted/browser-based YouTube 360 path, for readily available immersive video content.
- Use Content Bank entries and saved sessions to switch between prepared scenarios or media states.
- Use the Home layer and Home components to build simple on-screen launch tiles, menus, or section navigation.
- Use Triggers and Actions to link on-screen tiles, layer events, session loads, or other simple inputs to workspace, session, layer, WebView, and Content Bank actions.
- Use Home or Content Bank tiles to keep navigation simple, repeatable, and low-cognitive-load for the audience and operator.

### Authoring tools / content prep
- Source or create the 360 imagery/video before loading it into Igloo.
- Build accessibility into the source content itself, for example by using captions/subtitles in the source platform or burning subtitles into the prepared video.

### Alternative routes / integrations
- None

### Interaction model
- Use Canvas UI or the Home layer to launch the prepared session during the demo.
- Keep switching operator-led rather than promising automatic narrative logic.
- A light-touch interactive flow can be built natively by pressing tiles or simple icons that trigger sessions, layouts, active tiles, URLs, or layer actions.
- Use simple on-screen icons or trigger tiles over the scene to open supporting PDFs, swap layers, reveal web content, or move to another prepared workspace.
