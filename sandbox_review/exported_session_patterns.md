# Exported Session Patterns

Catalogued sessions: 5

Observed common patterns:
- exported sessions use a single `.iceSession` file with nearby thumbnails/assets
- immersive routes commonly rely on `PerspectiveExtraction`, sometimes with `InputTranslation` first
- `WebView` layers use `URL` rather than `FilePath`
- `YouTube` layers use a localhost `icetube` route
- realtime engine sessions use `Spout` plus `SenderName`, and often also include `ExePath`
- exported sessions often include a `Default Layout` region set and `ta` block

- ArchVis: Spout via ExePath (immersive media)
