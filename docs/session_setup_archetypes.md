# Session Setup Archetypes

This note captures the practical setup categories the builder now uses when planning room composition from briefs and local example sessions.

## Why this exists

Content type alone is too weak.

A `WebView` can mean:

- a three-wall Miro canvas
- a single supporting reference site
- a dashboard wall in a review room
- an interactive tour web app

The builder now tries to classify the intended session shape first, then place content into roles that make sense for that setup.

## Current archetypes

- `immersive_hero_360`
  - One main 360 or immersive hero scene with smaller support panels or later switch states.
  - Example references: `London - AirPano VR`, `Around the World - AirPano VR`, `Space Journey`, `Underwater`.

- `three_wall_canvas`
  - One collaboration or board surface stretched intentionally across the three main walls.
  - Example reference: `Miro - Displays`.

- `three_wall_dashboard`
  - One meaningful source per wall, such as dashboard, model, and support site.
  - Example references from review screenshots: `ESA dashboard`, `Aldar dashboard`.

- `three_wall_presentation`
  - One presentation surface stretched wide across the room.
  - Example reference: `Slides.com example`.

- `immersive_model_viewer`
  - One dominant model or pano viewer route, with minimal support panels.
  - Example references: `Matterport`, `Model Viewer - Simple`, `ArchVis`.

- `interactive_tour_webapp`
  - One web-based interactive tour or guided experience rather than a generic flat website.
  - Example references: `Streetview`, `Avatour`, `Blockade Labs - Skybox`.

- `interactive_training_webapp`
  - A specialist training or lesson platform with interaction logic inside the app.
  - Example reference: `CenarioVR`.

- `content_bank_gallery`
  - A prepared set of content states switched through Content Bank or a launcher flow.
  - Example references from screenshots: `Energy Content Bank`, `Hershey`.

- `operator_reference_surface`
  - A supporting reference or documentation surface, not the hero content.
  - Example reference: `ICE Documentation`.

## Practical use

These archetypes currently influence:

- brief planning
- search bias
- local sample-session fallback selection
- room preview composition
- session export layout defaults

They are meant to stay practical and example-led, not theoretical.
