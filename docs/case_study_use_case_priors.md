# Website And Public Use-Case Priors

This note records the practical workflow priors taken from Igloo's public website and public social surfaces.

These priors do not count as runtime truth.
They are used only to help Phase 1 recognise common real-world Igloo scenarios more accurately.

## Reviewed public sources

- [Igloo case studies](https://www.igloovision.com/case-studies)
- [Igloo video gallery](https://www.igloovision.com/about/video-gallery)
- public LinkedIn post previews for Igloo Vision

## Structure types repeatedly shown

- immersive workspaces
- retrofit rooms
- cylinders
- cubes
- domes
- CAVEs
- custom builds

## Repeating use-case archetypes

### Education

- immersive classrooms
- phygital teaching spaces
- repeatable lesson delivery
- virtual field trips
- teacher training
- tailored learning environments

### Heritage / experience / exhibition

- heritage storytelling
- portal-to-place experiences
- archival imagery and guided narrative
- museum or exhibition-style interpretation

### AECO / visualisation / collaboration

- BIM and model review
- markups and comparison layouts
- collaborative design decisions
- hybrid review with remote participants

### Training / simulation

- safety and onboarding
- classroom or facilitated simulation
- collaborative scenario review

### Corporate / brand / workshop use

- operator-led presentations
- hybrid workshops
- dashboards and live content alongside prepared media
- immersive storytelling in demo or briefing environments

## Video-gallery workflow signals

The public video gallery reinforces several concrete workflow patterns:

- standard video adaptation for Igloo
- 360 image workflows
- Google Street View / location-based immersive routes
- Google Slides 360 presentations
- Power BI in Igloo
- Unity plug-in workflows
- Revizto
- Igloo Control
- Warp & Blend
- VR collaboration

## How Phase 1 should use these priors

- improve route suggestions
- improve workflow-fit judgment
- improve control-fit judgment
- recognise when a brief matches a common Igloo deployment pattern

These priors must not:

- override runtime API truth
- override product documentation
- turn marketing examples into hard support claims
